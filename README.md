## 2. 方式一：通过安装包使用

安装包位置：

- `02_安装包_v4.2/PTA自动代码写入工具-安装包-v4.2.exe`

### 2.1 安装步骤

1. 打开 `02_安装包_v4.2` 文件夹。
2. 双击 `PTA自动代码写入工具-安装包-v4.2.exe`。
3. 按提示完成安装。
4. 安装完成后，可从桌面快捷方式或开始菜单启动软件。

### 2.2 首次运行前建议

为了保证功能正常，建议先确认以下几点：

- 电脑已安装 `Microsoft Edge` 或 `Google Chrome`。
- 电脑可以正常访问 `https://pintia.cn/`。
- 首次运行时如果需要下载浏览器驱动，网络应保持可用。

### 2.3 安装后如何使用

进入软件后，界面主要分为以下几部分：

- 顶部状态区
  - `登录状态`
  - `浏览器状态`
  - `本地登录态`
- 中间左侧 `代码编辑工作区`
- 中间右侧 `当前状态面板` 与 `运行日志`
- 底部操作按钮区

底部按钮说明：

- `打开浏览器并手动登录`
  - 打开浏览器，进入 PTA 登录页。
  - 你需要自己在浏览器里登录账号。
  - 登录成功后，工具会记录本地登录态。

- `仅写入（需已登录）`
  - 适合已经登录且浏览器中已打开题目页面的情况。
  - 点击后会把代码写入到 PTA 题目编辑器。

- `切换账号`
  - 会关闭当前浏览器并清理本地登录态。
  - 之后重新打开浏览器，便于登录另一个账号。

- `一键开始`
  - 推荐的完整流程按钮。
  - 在登录态可用时，会优先复用现有状态并继续执行写入流程。

## 3. 方式二：通过源代码使用

源代码位置：

- `01_源代码_v4.2`

这个目录里已经包含：

- 核心源码
- 图标和 logo 资源
- 默认代码文件
- 依赖清单 `requirements.txt`
- 一键启动脚本 `启动GUI.bat`
- 基础说明文件

### 3.1 运行要求

建议环境：

- Windows 系统
- Python 3.12 或更高版本
- 已安装 Edge 或 Chrome

### 3.2 最简单的源码运行方式

1. 打开 `01_源代码_v4.2` 文件夹。
2. 双击 `启动GUI.bat`。
3. 首次运行时，脚本会自动：
   - 创建 `.venv` 虚拟环境
   - 安装依赖
   - 启动图形界面

### 3.3 手动运行方式

如果你习惯手动命令行方式，可以按下面步骤操作：

```powershell
py -m venv .venv
.\.venv\Scripts\activate
python -m pip install -r requirements.txt
python gui_app.py
```

### 3.4 源码目录主要文件说明

- `gui_app.py`
  - 图形界面主程序
- `pta_auto_type.py`
  - 自动写入的核心逻辑
- `get_cookies.py`
  - 登录态获取与保存逻辑
- `code.txt`
  - 默认代码文件
- `cookies.json`
  - 首次登录成功后自动生成的本地登录态文件
- `favicon.ico`
  - 程序图标
- `logo.png`
  - 程序 logo

## 4. 软件标准使用流程

如果你是第一次使用，推荐按这个顺序操作：

### 4.1 第一步：准备代码

你有两种方式准备代码：

1. 直接把代码粘贴到 `代码编辑工作区`。
2. 把代码保存到 `code.txt`，然后点击 `加载 code.txt`。

编辑区上方按钮说明：

- `加载 code.txt`
  - 把当前目录下的 `code.txt` 内容加载到编辑区
- `保存到 code.txt`
  - 把编辑区当前内容保存回 `code.txt`
- `清空输入区`
  - 清空编辑区中的内容

### 4.2 第二步：登录 PTA

点击：

- `打开浏览器并手动登录`

随后会发生：

1. 工具自动打开浏览器。
2. 浏览器进入 PTA 登录页。
3. 你需要在浏览器中手动完成登录。
4. 登录成功后，工具会检测并保存登录态。

登录完成后，你可以观察顶部状态区：

- `登录状态`
- `浏览器状态`
- `本地登录态`

如果显示已登录、浏览器可用、本地登录态已检测到，就说明可以继续。

### 4.3 第三步：进入题目页面

你需要在浏览器中打开目标 PTA 题目页面，并确保页面中的代码编辑区域已经可见。

### 4.4 第四步：执行写入

根据你的使用场景选择：

- 如果当前浏览器已经打开题目页面，点击 `仅写入（需已登录）`
- 如果想让工具按完整流程自动处理，点击 `一键开始`

执行过程中请观察右侧：

- `当前状态面板`
- `运行日志`

日志区会显示当前执行步骤、成功提示或错误原因。

## 5. 顶部状态区怎么理解

顶部有三个状态项：

### 5.1 登录状态

用于表示当前是否具备继续操作的登录条件。

常见情况：

- `未登录`
- `已保存登录态`
- `可直接写入`

### 5.2 浏览器状态

用于表示工具是否检测到可用浏览器会话。

常见情况：

- `浏览器未连接`
- `浏览器已连接`

### 5.3 本地登录态

用于表示本地是否已经存在保存下来的登录 Cookie。

常见情况：

- `未检测到`
- `已检测到`

## 6. 常见使用场景

### 6.1 场景一：第一次使用

推荐流程：

1. 启动软件
2. 粘贴代码
3. 点击 `打开浏览器并手动登录`
4. 浏览器中登录 PTA
5. 打开题目页面
6. 点击 `仅写入（需已登录）` 或 `一键开始`

### 6.2 场景二：上次已经登录过

如果顶部显示本地登录态已经存在：

1. 启动软件
2. 加载或粘贴代码
3. 打开目标题目页面
4. 直接尝试 `一键开始` 或 `仅写入（需已登录）`

如果这是别人发给你的干净分享包，首次启动时通常不会直接显示“本地登录态已检测到”，这属于正常情况。

### 6.3 场景三：需要更换账号

推荐流程：

1. 点击 `切换账号`
2. 等工具关闭当前浏览器和旧登录态
3. 再点击 `打开浏览器并手动登录`
4. 登录新账号

## 7. 常见问题排查

### 7.1 点了登录没反应

请检查：

- 电脑是否安装了 Edge 或 Chrome
- 浏览器是否被安全软件拦截启动
- 网络是否正常

### 7.2 首次运行很慢

这是正常现象，常见原因有：

- 首次安装 Python 依赖
- 首次下载浏览器驱动
- 首次初始化 GUI 资源

### 7.3 提示没有检测到浏览器

请先安装以下任意一个浏览器：

- Microsoft Edge
- Google Chrome

### 7.4 已登录但仍无法写入

请检查：

- 当前浏览器是否真的打开了 PTA 题目页面
- 题目页面中的代码编辑器是否已加载完成
- 是否在错误的标签页中执行写入

### 7.5 日志里提示登录超时

说明在规定时间内没有检测到登录成功。

建议：

1. 重新点击 `打开浏览器并手动登录`
2. 尽快在浏览器里完成登录
3. 如有验证码或二次验证，请手动完成

### 7.6 源码运行时报依赖缺失

请在源码目录重新执行：

```powershell
python -m pip install -r requirements.txt
```

## 8. 建议的使用习惯

- 使用前先确认目标题目页面已经打开。
- 代码尽量先在本地整理好，再粘贴到工具里。
- 若长期使用，建议保留 `code.txt` 方便反复加载。
- 如果账号异常或登录态混乱，优先使用 `切换账号` 重置环境。
